# Skill: 内容完整性检查

## 核心原则

**推送的内容必须与源文件一致**

不能截断、不能省略、不能假设用户会自己去看文件。

---

## 内容长度标准

| 长度 | 处理方式 | 说明 |
|------|---------|------|
| < 50行 | **完整推送** | 直接推送全部内容 |
| 50-100行 | **完整推送** | 直接推送全部内容 |
| 100-200行 | **分段推送** | 分2-3段推送，每段说明 |
| > 200行 | **概要+链接** | 推送概要 + 文件链接 + 说明 |

---

## 推送前检查清单

### 1. 统计源文件
```bash
# 统计行数
line_count=$(wc -l < 文件路径)

# 统计字数
char_count=$(wc -c < 文件路径)

echo "文件统计: $line_count 行, $char_count 字符"
```

### 2. 决定推送策略
```bash
if [ $line_count -lt 100 ]; then
    strategy="完整推送"
elif [ $line_count -lt 200 ]; then
    strategy="分段推送"
else
    strategy="概要+链接"
fi
```

### 3. 执行推送
根据策略推送内容。

### 4. 验证推送
```bash
# 统计实际推送的行数
pushed_lines=$(echo "$推送内容" | wc -l)

# 对比
if [ $pushed_lines -ge $line_count ]; then
    echo "✅ 推送完整"
else
    echo "⚠️ 推送不完整: $pushed_lines/$line_count 行"
fi
```

## 推送策略详情

### 策略A: 完整推送 (<100行)

**适用**: 日报、短报告、推荐

**操作**:
1. 读取完整文件
2. 完整推送到对话
3. 确认用户收到

**示例**:
```
# AI日报 | 2026年3月6日

## 头条新闻
1. ...
2. ...
...

（完整内容）
```

---

### 策略B: 分段推送 (100-200行)

**适用**: 长报告、详细分析

**操作**:
1. 分段读取文件
2. 逐段推送
3. 每段说明"第X部分，共Y部分"

**示例**:
```
📄 长报告 - 第1部分（共3部分）

...（前1/3内容）

---
⏳ 正在发送下一部分...
```

---

### 策略C: 概要+链接 (>200行)

**适用**: 非常长的文档、技术规范

**操作**:
1. 提取概要（标题、要点）
2. 推送概要
3. 提供文件链接
4. 说明完整内容位置

**示例**:
```
📄 技术规范概要

## 主要内容
1. 架构设计
2. API接口
3. 部署指南

⚠️ 完整内容较长（250行），请查看：
- GitHub: [链接]
- 或回复"发送完整内容"获取全文
```

## 验证方法

### 方法1: 行数对比
```bash
source_lines=$(wc -l < source.md)
pushed_lines=$(echo "$pushed" | wc -l)

if [ $pushed_lines -ge $source_lines ]; then
    echo "✅ 完整"
else
    echo "❌ 不完整: $pushed_lines/$source_lines"
fi
```

### 方法2: 字数对比
```bash
source_chars=$(wc -c < source.md)
pushed_chars=$(echo -n "$pushed" | wc -c)

# 允许5%的误差（格式转换）
min_chars=$((source_chars * 95 / 100))

if [ $pushed_chars -ge $min_chars ]; then
    echo "✅ 完整"
else
    echo "❌ 不完整"
fi
```

### 方法3: 关键内容检查
```bash
# 检查关键章节是否存在
if echo "$pushed" | grep -q "## 头条新闻"; then
    echo "✅ 关键章节存在"
else
    echo "❌ 关键章节缺失"
fi
```

## 用户确认

推送后必须确认：

```
完整内容已推送，请确认：
1. 是否收到？
2. 内容是否完整？
3. 格式是否正确？

如有问题，请立即告诉我。
```

---

## 错误处理

### 发现推送不完整
1. **立即道歉**: "抱歉，内容不完整"
2. **补发缺失**: 发送缺失部分
3. **解释原因**: 说明为什么不完整
4. **确认完整**: 确保用户收到完整内容

### 用户反馈不完整
1. **立即响应**: "收到，立即补发"
2. **使用正确策略**: 根据长度选择策略
3. **分段确认**: 每段发送后确认
4. **最终确认**: "请确认现在完整了"
