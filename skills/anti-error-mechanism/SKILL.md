# Skill: 防错机制 - 确保不再重复犯错

## 版本: 1.0
## 创建日期: 2026-03-06
## 创建原因: 一周8个错误，反复犯同样问题

---

## 核心问题

为什么创建了8个错误记录和5个修复Skill，还是继续犯错？

**答案**: 因为没有强制执行力。Skill只是参考，不是规则。

---

## 真正的解决方案

### 1. 强制前置检查（必须执行）

**每次任务前必须运行**:
```bash
bash scripts/mandatory-pre-check.sh
```

**不执行后果**: 任务不能开始

**检查内容**:
- 读取标准流程
- 检查今日错误
- 确认理解核心原则
- 任务类型特定要求

---

### 2. 简化流程（减少决策点）

**使用简化版清单**: `SIMPLE_CHECKLIST.md`

**原则**:
- 不打勾不能继续
- 用户确认才算完成
- 出错立即停止

---

### 3. 自动验证（不依赖人工）

**推送后自动运行**:
```bash
bash scripts/auto-verify-push.sh
```

**自动检查**:
- 文件行数
- 文件大小
- 长内容警告

---

### 4. 用户确认强制步骤

**推送后必须**:
```
完整内容已推送，请确认：
1. 是否收到？
2. 内容是否完整？
3. 格式是否正确？

回复确认后任务才算完成。
```

**不确认不算完成**

---

## 执行顺序（不可跳过）

```
1. 任务开始
   ↓
2. 强制前置检查 (mandatory-pre-check.sh)
   ↓
3. 执行任务
   ↓
4. 按SIMPLE_CHECKLIST逐项检查
   ↓
5. 推送到对话框（完整内容）
   ↓
6. 自动验证 (auto-verify-push.sh)
   ↓
7. 用户确认
   ↓
8. Git提交推送
   ↓
9. 任务完成
```

## 关键改变

| 之前 | 现在 |
|------|------|
| Skill是参考 | Skill是强制规则 |
| 人工检查 | 自动验证 |
| 我以为完成 | 用户确认才算完成 |
| 复杂CHECKLIST | 简化版，不打勾不能继续 |
| 出错后记录 | 前置预防 |

---

## 每日执行

### 早上
1. 运行: `bash scripts/daily-self-check.sh`
2. 处理发现的问题
3. 阅读昨天的错误

### 每次任务
1. 运行: `bash scripts/mandatory-pre-check.sh`
2. 按SIMPLE_CHECKLIST执行
3. 用户确认后才算完成

### 晚上
1. 回顾今天的执行
2. 更新错误记录（如有）
3. 优化流程

## 承诺

使用这个防错机制后，我承诺：

1. **每次任务前运行前置检查**
2. **严格按照简化清单执行**
3. **用户确认才算完成**
