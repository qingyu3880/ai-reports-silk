# 错误复盘与防错系统

## 一、所有出过的错误清单

### 1. 定时任务配置错误（3月5日 8:00前）
**错误：** `delivery.channel: "whatsapp"` + `sessionTarget: "isolated"` 导致 "Unsupported channel: whatsapp"
**根因：** 任务配置使用了错误的channel和session类型
**修复：** 改为 `sessionTarget: "main"` + `delivery: {}`

### 2. 定时任务未执行（3月5日 9:00）
**错误：** 9:00 AI日报未推送

**修复：** 手动生成并推送

### 3. 音乐推荐格式错误（3月5日 8:35）

**根因：** 任务提示词未强制要求格式检查
**修复：** 更新任务配置，加入详细格式要求

### 4. 网站数据不同步（3月5日 9:30）
**错误：** 网站没有音乐专栏
**根因：** `sync_reports.py` 未收集music和movies数据
**修复：** 修改脚本，添加music和movies收集逻辑

### 5. 网站未更新（3月5日 9:40）

**根因：** 两个目录未同步

### 6. 图书推荐未执行（3月5日 17:00）

**状态：** 待补救

## 二、错误模式分析

| 模式 | 次数 | 根本原因 |
|------|------|----------|
| 配置错误 | 2次 | 对cron机制理解不深 |
| 执行遗漏 | 2次 | 收到消息未立即处理 |
| 路径/同步错误 | 2次 | 对部署架构不熟悉 |

**核心问题：** 缺乏**主动检查**机制，总是被动等待报错

## 三、防错系统（立即执行）

### 第一层：任务触发时（自动）
- [ ] 收到定时任务消息 → **立即执行**，不延迟
- [ ] 执行前检查：今天是否已有文件
- [ ] 执行后检查：文件是否生成且大小>1KB

### 第二层：每小时自查（18:00/19:00/20:00）
- [ ] 检查今天各目录文件生成情况
- [ ] 发现遗漏立即补做

### 第三层：任务完成后验证
- [ ] 推送到对话后，确认用户收到
- [ ] 运行sync_reports.py同步网站
- [ ] 检查网站数据是否更新

### 第四层：日终复盘（每天21:00）
- [ ] 检查当天所有任务完成状态
- [ ] 记录异常，更新本文件

## 五、关键原则

1. **收到即执行**：定时任务消息不积压，立即处理
2. **文件即证据**：没有文件=没有完成
3. **同步即完成**：不推送到网站=任务未完成
4. **用户确认**：用户未收到=任务未完成
