# 简化版任务执行清单（不可跳过）

## 使用说明
1. 每次任务前运行: `bash scripts/mandatory-pre-check.sh`
2. 严格按照本清单执行
3. 每完成一项打勾，不完成不能继续
4. 最后用户确认才算完成

---

## 内容生成任务（日报/推荐/报告）

### 阶段1: 生成
- [ ] 运行前置检查脚本
- [ ] 生成内容保存到文件
- [ ] **验证**: 文件存在且大小>1KB

### 阶段2: 用户交付（关键）
- [ ] **统计行数**: `wc -l < 文件`
- [ ] **完整推送到对话框**
  - 短内容(<100行): 一次推送全部
  - 长内容(>100行): 分段推送，每段说明
- [ ] **验证推送**: 对比行数是否一致
- [ ] **用户确认**: 询问"是否收到完整内容？"

### 阶段3: 持久化
- [ ] Git add
- [ ] Git commit
- [ ] Git push
- [ ] 验证推送成功: `git log origin/main..master` 为空

### 阶段4: 完成确认
- [ ] 用户明确表示"收到"或开始讨论内容
- [ ] 任务才算真正完成

## 技能安装任务

### 阶段1: 准备
- [ ] 运行前置检查脚本
- [ ] 明确要安装的技能

### 阶段2: 安装
- [ ] 尝试安装
- [ ] **验证真实存在**: `ls skills/技能名/`
- [ ] **验证可调用**: 测试工具是否可用

### 阶段3: 验证
- [ ] 不依赖ClawHub的"成功"报告
- [ ] 亲自验证文件系统
- [ ] 记录真实安装的技能

---

## 同步推送任务

### 阶段1: 检查
- [ ] 运行前置检查脚本
- [ ] 检查未推送提交: `git log origin/main..master`

### 阶段2: 推送
- [ ] 执行推送
- [ ] 验证推送成功

### 阶段3: 同步
- [ ] 运行sync脚本
- [ ] 验证reports.json更新

---

## 强制规则

1. **不打勾不能继续**: 每一项必须完成并打勾
2. **用户确认必须**: 没有用户确认，任务不算完成
3. **出错立即停止**: 发现问题不继续，先修复
4. **记录所有错误**: 每个错误都要写成ERROR_xxx.md

---

## 常见错误速查

| 错误 | 原因 | 解决 |
|------|------|------|
| 只推送到GitHub | 忘记用户交付阶段 | 严格按照阶段2执行 |
| 只推送概要 | 没有验证行数 | 推送前统计，推送后对比 |
| 用户说没收到 | 没有用户确认步骤 | 必须询问并等待确认 |
| 虚假完成报告 | 没有真实验证 | 验证文件系统，不依赖日志 |
